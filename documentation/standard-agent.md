# Standard Legal AI Agent

**File:** `app/api/agent/route.ts`
**Transport:** SSE (Server-Sent Events) over HTTP POST to `/api/agent`
**Runtime:** Node.js (Next.js route, `maxDuration: 120s`)
**Default provider:** Gemini (configurable to Anthropic or OpenAI)

---

## What It Does

The standard agent is Verdictu's main legal research pipeline. It takes a user question and jurisdiction, identifies the applicable laws, searches the web for statute text and case law, validates that the search stayed on target, then streams a structured legal analysis back to the client. It always runs the full pipeline ‚Äî search is never skipped.

---

## Step-by-Step Pipeline

### Step 1 ‚Äî Intake
**SSE event:** `intake`

Emits immediately with the jurisdiction, mode, and a 120-character preview of the message. No AI call is made here ‚Äî this is a UI signal so the chat can show "starting" state.

---

### Step 2 ‚Äî Law Identification (Turn 1)
**SSE event:** `identifying` ‚Üí `laws_found`
**Prompt:** `lawIdentificationPrompt(jurisdiction)` from `lib/ai/prompts.ts`
**Max tokens:** 1,200
**Mode:** JSON (non-streaming)

The agent calls the AI provider with a senior-legal-researcher system prompt and asks it to identify every law, statute, regulation, and article that applies to the question.

The model returns a JSON object:
```json
{
  "laws": [
    {
      "name": "GDPR",
      "citation": "Art. 17(1) GDPR (Regulation 2016/679)",
      "relevance": "primary",
      "confidence": 0.95,
      "applies_because": "User is requesting erasure of personal data"
    }
  ],
  "searchQueries": ["GDPR right to erasure Art 17 conditions", "..."],
  "legalDomain": "gdpr",
  "jurisdictionConfirmed": "EU"
}
```

Laws are ranked by relevance (primary ‚Üí secondary ‚Üí supplementary). Confidence 0.9+ = certain, 0.6‚Äì0.8 = probable.

This step is **non-fatal** ‚Äî if parsing fails the agent continues with empty laws and a fallback search query built from the raw message.

---

### Step 3 ‚Äî Web Search (Turn 2)
**SSE events:** `searching` (per query) ‚Üí `search_results` (per query) ‚Üí `sources_ranked`
**Search provider:** Tavily API (if `TAVILY_API_KEY` set) or DuckDuckGo Lite (fallback)
**Queries run:** Up to 5, generated by the AI in Step 2

Searches are run **sequentially** (not in parallel) so the UI can show per-query progress. Each query fetches up to 5 results. Results are deduplicated by URL ‚Äî if the same URL appears in multiple queries, only the first occurrence is kept.

After all queries finish, `sources_ranked` is emitted with the total count and which search engine was used.

**Search query strategy:** The AI generates queries optimized for finding statute text and case law ‚Äî mixing the law name, article number, and topic keywords.

---

### Step 4 ‚Äî Alignment Check (Turn 2.5)
**SSE events:** `aligning` ‚Üí `alignment_result`
**Prompt:** `alignmentCheckPrompt()` from `lib/ai/prompts.ts`
**Max tokens:** 300
**Mode:** JSON (non-streaming)

This is a quality-control layer that runs after law identification and search, before synthesis. It sends the AI the original question plus the identified laws and domain, and asks: "Is the agent on track to answer what was actually asked?"

The model returns:
```json
{
  "aligned": true,
  "originalIntent": "User wants to know if they can request deletion of their data",
  "correctionNote": ""
}
```

If `aligned` is `false`, the `correctionNote` is prepended to the synthesis message as `‚öÝÔ∏è ALIGNMENT CORRECTION: ...` to steer the model back to the user's actual question.

This step is **non-fatal** ‚Äî if it errors, synthesis proceeds without correction.

---

### Step 5 ‚Äî Synthesis (Turn 3)
**SSE events:** `synthesizing` ‚Üí `delta` (one per token)
**Prompt:** `synthesisPrompt(jurisdiction, mode, citationEnabled)` from `lib/ai/prompts.ts`
**Max tokens:** 4,000
**Mode:** Streaming

The full user message is assembled from:
1. The original question (with optional `‚öÝÔ∏è ALIGNMENT CORRECTION` prefix)
2. Any attached documents (up to 8,000 chars each)
3. The identified laws with relevance and confidence
4. The top 10 web search results (up to 800 chars per snippet)

This assembled message is sent to the AI with the synthesis system prompt. Tokens are streamed to the client as `delta` events in real time.

**Output structure varies by mode:**

**General mode** ‚Äî structured legal analysis:
- `## Summary` ‚Äî direct, definitive answer
- `## Legal Basis` ‚Äî specific statutes and citations
- `## Analysis` ‚Äî rule applied to facts, uncertainty flagged
- `## Practical Implications` ‚Äî what to do next
- `## Disclaimer`

**Compare mode** ‚Äî document comparison:
- `## Document Overview`
- `## Conflicts` ‚Äî with quotes from each document
- `## Gaps`
- `## Risk Assessment` ‚Äî table with HIGH/MEDIUM/LOW severity
- `## Recommendations`
- `## Disclaimer`

**Draft mode** ‚Äî document drafting:
- Formal legal language for the jurisdiction
- `[PLACEHOLDER: description]` for user-specific details
- Disclaimer at end

If `citationEnabled` is true, inline citations `[1]`, `[2]` are used and a **Sources** section is appended.

---

### Step 6 ‚Äî Follow-up Questions (Turn 4)
**SSE events:** `follow_up_generating` ‚Üí (included in `done`)
**Prompt:** `followUpPrompt(jurisdiction)` from `lib/ai/prompts.ts`
**Max tokens:** 400
**Mode:** JSON (non-streaming)

Sends the original question + first 1,500 chars of the answer summary. The model returns 2‚Äì4 clarifying questions targeting specific facts that would change the legal outcome.

This step is **non-fatal** ‚Äî if it errors, `followUpQuestions` is an empty array in the `done` event.

---

### Step 7 ‚Äî Done
**SSE event:** `done`

Final event with:
```json
{
  "sources": [...],        // top 12 sources (title, url, domain)
  "laws": [...],           // identified laws from Step 2
  "followUpQuestions": [], // 2‚Äì4 questions from Step 6
  "wordsInAnswer": 850     // word count of the streamed answer
}
```

## SSE Event Reference

| Event | When emitted | Key fields |
|---|---|---|
| `intake` | Start of pipeline | `jurisdiction`, `mode`, `preview` |
| `identifying` | Before law identification | `message` |
| `laws_found` | After law identification | `laws`, `domain`, `jurisdiction`, `count` |
| `searching` | Before each search query | `query`, `index`, `total` |
| `search_results` | After each search query | `query`, `count`, `sources` (top 3) |
| `sources_ranked` | After all queries done | `total`, `searchEngine` |
| `aligning` | Before alignment check | ‚Äî |
| `alignment_result` | After alignment check | `aligned`, `originalIntent`, `corrected` |
| `synthesizing` | Before streaming synthesis | `message` |
| `delta` | Each streamed token | `text` |
| `follow_up_generating` | Before follow-up call | ‚Äî |
| `done` | Pipeline complete | `sources`, `laws`, `followUpQuestions`, `wordsInAnswer` |
| `error` | Any unhandled error | `message` |

## Provider Support

The agent supports three AI providers, selected by the `provider` field in the request body:

| Provider | Streaming | JSON mode | Default model |
|---|---|---|---|
| `gemini` (default) | `generateContentStream` | Via `generationConfig` | `gemini-2.5-flash` |
| `anthropic` | `messages.stream` | Via `betas` or prompt | `claude-sonnet-4-6` |
| `openai` | `chat.completions.create(stream: true)` | Via `response_format` | `gpt-4o` |

---

## Request Body

```typescript
{
  message: string;           // required
  jurisdiction: string;      // e.g. "EU", "DK", "DE" ‚Äî defaults to "EU"
  mode?: "General" | "Compare" | "Draft";  // defaults to "General"
  attachments?: Array<{ filename: string; text: string }>;
  citationEnabled?: boolean; // defaults to true
  provider?: "gemini" | "anthropic" | "openai";  // defaults to "gemini"
  model?: string;            // overrides default model for the provider
}
```
